TotalWare API

Scan your files programmatically — no browser needed. Every request is authenticated with an API key. Head to the API Keys page to create one.

Base URL

http://127.0.0.1:5000

After deployment this becomes your domain (e.g. https://totalware.yourdomain.com).

Authentication

Send your API key in the request header. Either method works:

# Recommended
Authorization: Bearer tw_live_xxxxxxxxxxxx

# Ya
X-API-Key: tw_live_xxxxxxxxxxxx

An invalid or missing key returns 401 Unauthorized.

Scan a file

POST/api/v1/scan

Upload a file (multipart form). The response includes the verdict, risk score, and reasons.

FieldTypeRequiredDescription
filefileYesScan karne wali file (APK, EXE, PDF, ZIP, PCAP, etc.)
passwordtextNoPassword for an archive (zip/rar), if any

Example — cURL

curl.exe -X POST http://127.0.0.1:5000/api/v1/scan \
  -H "Authorization: Bearer tw_live_xxxxxxxxxxxx" \
  -F "file=@C:\path\to\sample.apk"

Example — Python

# pip install requests
import requests

url = "http://127.0.0.1:5000/api/v1/scan"
headers = {"Authorization": "Bearer tw_live_xxxxxxxxxxxx"}

with open("sample.apk", "rb") as f:
    r = requests.post(url, headers=headers, files={"file": f})

print(r.json())

Verdicts

VerdictScoreMeaning
MALICIOUS75–100Very likely malware — do not install or run
SUSPICIOUS50–74Suspicious — investigate further
POTENTIALLY UNWANTED25–49Some red flags, but probably not malware
CLEAN0–24No significant threat detected

Rate limits

Each key is limited to 30 requests / minute. Exceeding it returns 429 Too Many Requests along with retry_after_seconds.
